Los Dockers: 
Son encapsulamientos de código para que puedan ser enviados y recibidos y funcionen en todas partes. Se trata de tomar una foto (imagen) del proyecto en ese momento y enviar eso. 
Tienen similitud con las máquinas virtuales ya que cada contenedor es casi lo mismo en funcionamiento, pero en el fondo están construidos diferentes: 
Las máquinas virtuales agarran un servidor (que es en esencia un PC físico potente en algún data center) y dividen sus recursos físicos en diferentes porciones (usando una tecnología llamada hipervisor) para que en cada uno de estos se instalen diferentes sistemas operativos y puedan correr diferentes apps. En este caso están virtualizando el HARDWARE del PC. Lo malo de esto es que en cada división se tiene que instalar una version de sistema operativo diferente y esto puede requerir muchos recuros, imagina que tienes 5 debians montados en 5 máquinas virtuales, en verdad repites cosas que podrías solo guardar una vez.  

Por otro lado, los contenedores lo que hacen es que virtualizan el SISTEMA OPERATIVO. Lo que hace es que sube el Kernel de linux que es igual para todas las versiones de Linux, entonces, cuando crear un contenedor, lo unico que hace es traer las capas de linux extras necesarias para correr la app aprovechando el Kernel que es compartido, esto hace que sean super livianos. La app de Docker posee varios tipos da datos: 
1. El Dockerfile es un archivo de texto con instrucciones paso a paso para construir una imagen Docker. Define qué software instalar y cómo configurar la imagen.
2. La imagen es el resultado de construir un Dockerfile; es un archivo de solo lectura que contiene la app y el entorno necesario.
3. El contenedor es la instancia en ejecución creada a partir de esa imagen, donde corre la aplicación.



ARQUITECTURA: ----------------------------------------------------------------------------------------------------------------------------------
0. Daemon (Dockerd): el f goat, el corazón o motor de Docker. Está montado en el PC que tenga descargado Docker, como es un daemon siempre está corriendo en segundo plano, es como el que hace todo el trabajo sucio de enviar, recibir peticiones API REST, correr las imagenes, implementar los archivos. etc, para que los contenedores funcionen.

1. Host Docker: PC en dónde está instalado Docker (puede ser mi PC o un servidor [q es un super pc de alguien más]). Es el PC que da los recursos FÍSICOS para correr los contenedores.

2. Cliente Docker (CL: Command Line): Es una terminal en donde se pasan peticiones al Host, que luego el daemon lee y ejecuta. Peticiones como: "docker run". Las peticiones son API REST.
