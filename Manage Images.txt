Pasos para descargar una imagen de DockerHub: ------------------------------------------------------------------------------------------------------------------------------------------

0. Verificar si tengo docker: 

docker --version

1. Si hago: docker images, confirmo las imagenes que tengo descargadas (no debería tener ninguna).

2. docker pull hello-world

Docker busca esa imagen llamada hello-world en el registro oficial de imágenes Docker público llamado Docker Hub (hub.docker.com) y la descarga a tu máquina local si no la tienes ya.

2.2 ahora "docker images" muestra la imagen descargada:

REPOSITORY    TAG       IMAGE ID       CREATED       SIZE
hello-world   latest    d2c94e258dcb   2 years ago   13.3kB

3. Correr la imagen: "docker run hello-world": 

"Hello from Docker!

 1. The Docker client contacted the Docker daemon.

 3. The Docker daemon created a new container from that image which runs the
    executable that produces the output you are currently reading.
 4. The Docker daemon streamed that output to the Docker client, which sent it

Crear imagenes ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

0. crear mi archivo Dockerfile: 

FROM node:9.4.0-alpine
COPY app.js .
COPY package.json .
RUN npm install &&\
    apk update &&\
    apk upgrade
EXPOSE  8080
CMD node app.js


1. buildearlo con linux: 

"docker build . -t myimage:v1"

acá agarra el dockerfile y crea la imagen de mi archivo: (docker images:)

REPOSITORY    TAG       IMAGE ID       CREATED          SIZE
myimage       v1        034fe82dac28   13 seconds ago   77.8MB

MAGIA. 

2. Ahora que está creada la imagen la puedo ejecutar (es decir crear un contenedor): 

docker run -dp 8080:8080 myimage:v1 

curl localhost:8080 //esta cosa descargaba algo de una url en linux, lo que descargo es: 

"Hello world from b31dac61af2b! Your app is up and running!"

3. pararla con: "docker stop $(docker ps -q)" y confirmar si paró con: "docker ps"
